## 2️⃣ What We Did in **Phase 1 – Setup & Index Creation**

### 2.1 Code Files Created (see `src/ingestion/`)
| File | Purpose | Key Functions |
|------|---------|----------------|
| `document_loader.py` | Load PDFs from a directory | `load_documents(input_dir: str) -> List[Document]` – uses `SimpleDirectoryReader` from **LlamaIndex** |
| `chunker.py` | Split loaded documents into true semantic chunks | `chunk_documents(documents: List[Document]) -> List[BaseNode]` – uses `SemanticSplitterNodeParser` to cut text dynamically at topic breakpoints. |
| `embedder.py` | Provide an embedding model | `get_embedder() -> HuggingFaceEmbedding` – wraps **intfloat/e5‑large‑v2** (1024‑dimensional) |
| `build_index.py` | End‑to‑end pipeline that creates a FAISS vector store | `main()` – sets `Settings.embed_model`, loads docs, chunks them, builds `FaissVectorStore`, persists to `data/llamaindex_storage/` |

### 2.3 Rationale Behind Technology Choices (IEEE‑style justification)
- **Open‑source stack** – All components are free, reproducible, and can run on commodity hardware (CPU‑only). This satisfies the “cost‑effective” constraint in the SPEC.
- **LlamaIndex** – Provides a unified API for document loading, chunking, embedding, and vector‑store integration, dramatically reducing boiler‑plate and ensuring future extensibility (e.g., swapping out the embedder).
- **E5‑Large‑V2** – State‑of‑the‑art open‑source embedding model with strong performance on retrieval benchmarks (MTEB). It produces 1024‑dimensional vectors, which align perfectly with FAISS’s default inner‑product metric.
- **FAISS** – Industry‑standard similarity search library; works offline, has sub‑millisecond query latency on a laptop, and is widely cited in academic RAG papers.
- **True Semantic Chunking** – Instead of blindly slicing medical text every 512 words (which fragments meaning), we use `SemanticSplitterNodeParser`. It embeds every sentence and mathematically detects when the topic changes, ensuring chunks encapsulate complete medical thoughts natively.
- **Virtual environment & pinned `requirements.txt`** – Guarantees deterministic builds for reviewers and reproducibility for the conference submission.

## What Happens When You Run **`/execute 2`** (Phase 2 – RAG Pipeline)

In plain, non‑technical language, here’s what the system will do step by step:

1. **Turn on the brain** – The code loads a small, fast version of the BioMistral language model (a 4‑bit AI that can write answers). It runs on your regular CPU, so no special hardware is needed.

2. **Open the library of knowledge** – It reads the FAISS index we built in Phase 1. This index is just a very smart list of all the sentences from your medical PDFs, organized so we can find the right ones instantly.

3. **Match your question to the right text** – When you ask a medical question, the system first looks in the FAISS index and pulls the most relevant textbook passages.

4. **Ask the brain to answer** – The retrieved passages are handed to BioMistral, which writes a concise answer that is based only on those passages (so it can’t hallucinate unrelated facts).

5. **Remember the conversation** – (Later in this phase) we’ll add a tiny memory component that keeps track of what you asked before, so follow‑up questions like “What about the side‑effects?” make sense.

6. **Run a quick test** – The executor will automatically ask a sample question (e.g., “What are the symptoms of type 2 diabetes?”) and print the answer. If you see a sensible answer, the whole pipeline is working.

7. **Mark the work as done** – After the test passes, [STATE.md](cci:7://file:///home/mohi/Documents/MiniProject/Project_files/ClinRAG/.gsd/STATE.md:0:0-0:0) is updated to show that Phase 2 is verified and ready for the next steps.

**Why we do each of these things:**
- The BioMistral model gives us a free, open‑source AI that can run on a laptop.
- FAISS lets us search millions of sentences in a fraction of a second.
- Connecting the two makes sure the AI only answers using trusted medical text.
- The conversation memory makes the chatbot feel natural.
- The test proves everything works before we move on.
